类实质上是封装对象属性和行为的载体，而对象是类抽象出来的一个实例

具有相同属性和行为的一类实体被称为类

面向对象的程序设计有以下特点：封装性、继承性和多态性

#### 封装

将对象的属性和行为封装起来，其载体就是类，类通常对客户隐藏其实现细节

#### 继承

一个类处于继承体系中，它既可以是其他类的父亲，为其它类提供属性和行为，也可以是其他类的子类，集成父类的属性和方法

#### 多态

将父类对象应用于子类的特征就是多态



#### 类

##### 成员变量

对象的属性称为成员变量，成员变量可以是任意类型，整个类中都是成员变量作用的范围

##### 成员方法（类似于函数）

```java
权限修饰符 返回值类型 方法名(参数类型 参数名){
	……
	return 返回值;
}
```

##### 权限修饰符

java的权限修饰符主要包括：private、public、protected

**如果成员变量或成员方法被设为private**：该成员变量只能在本类中使用，在子类中是不可见的，并且对其他的类也是不可见的

**如果成员变量或成员方法被设为public**：除了可以在本类中使用这些数据，还可以在子类和其他包的类中使用这些数据



**如果一个类的访问权限被设置为private**：这个类将隐藏其内的所有数据，以免用户直接访问它

**如果一个类的访问权限被设置为public**：类中的数据可以被子类或者其他包中的类使用

**如果一个类的访问权限被设置为protected**：那么只有本包内的该类的子类或其他类可以访问此类中的成员变量和成员方法



|    访问包位置    | 类修饰符 | 类修饰符  | 类修饰符 |
| :--------------: | :------: | :-------: | :------: |
|                  | private  | protected |  public  |
|       本类       |   可见   |   可见    |   可见   |
| 同包其他类或子类 |  不可见  |   可见    |   可见   |
| 其他包的类或子类 |  不可见  |  不可见   |   可见   |

##### 局部变量

在成员方法中定义一个变量，那么这个变量被称为局部变量，局部变量是在方法被执行时创建，在方法执行结束时被销毁，局部变量在使用时必须进行复制操作或被初始化，否则会出现编译错误

##### 局部变量的有效范围

可以将局部变量的有效范围成为变量的作用域，局部变量的有效范围从该变量的声明开始到该变量的结束位置

##### 静态变量和静态方法

被声明为static的变量和方法被称为静态成员。静态成员属于类所有，区别于个别对象，可以在本类或其它类使用类名和‘ . ’运算符调用静态成员

```java
 public class Cust{
 	static int count = 0;
 	String name;
 	public Cust(String name){
 		this.name = name;
 		count++;
 	}
 	
 	public static void main(String[] args){
 		Cust c1 = new Cust("Tom");
 		System.out.print(Cust.count + c1.name);
 		Cust c2 = new Cust("zhangsan");
 		System.out.print(Cust.count + c2.name);
 		Cust c3 = new Cust("dandan");
 		System.out.print(Cust.count + c3.name);
 	}
 }
 
 //输出
 1 Tom
 2 zhangsan
 3 dandan
```

count是一个静态的，是所有的Cust共有的属性，每新增一个顾客就count++，因此count可以用来计算Cust的总数

如果在执行类时，希望先执行类的初始化动作，可以使用static定义一个静态区域，这块区域称为静态代码块，当类文件被执行时，会首先执行static块中的程序，并且只会执行一次。

```java
public class 类名{
	static{
		……
	}
}
```

注意点：

* 静态方法中不可以使用this关键字
* 静态方法中不可以直接调用非静态方法
* 局部变量不可以使用static关键字进行声明
* 主方法必须使用static关键字进行声明



##### 类的主方法

主方法是类的入口点，它定义了程序从何处开始，主方法提供对程序流向的控制，java编译器通过主方法来执行程序

```java
//主方法语法
public static void main(String[] args){
	……
}
```

* 主方法是静态的，因此如要直接在主方法中调用其他方法，则该方法也必须是静态的
* 主方法没有返回值
* 主方法的形参为数组，其中args[0]~args[n]分别代表程序的第一个参数到第n个参数，可以使用args.length获取参数的个数



##### 对象

```java
Test test = new Test();
Test test = new Test("a");
```

Test是类名，用new来创建对象操作符

test是创建的Test对象，设置为a，a是构造方法的参数

每个对象都是相互独立的，在内存中占据独立的内存地址，并且每个对象都具有自己的生命周期，当一个对象的生命周期结束时，对象就变成垃圾，有Java虚拟机自带的垃圾回收机制处理，不能再被使用



###### 访问对象的属性和行为

使用new操作符创建一个对象后，可以使用“对象.类成员”来获取对象的属性和行为



###### 对象的引用

```java
//对象的引用
类名 对象的引用变量
//例如：(people类的引用)
People tom;
//通常一个引用不一定需要有一个对象与其相关联，引用与对象相关联的语法如下
People tom = new People();
```

上述代码中，其实真正的对象是new People()这段代码，为了方便开发者保存、调用对象，于是创建了一个People类型、名叫tom 的引用变量。实际上tom只是一段内存地址，用于标记new People()对象在内存中的位置，简单来说，tom是new People()的代理人，那么new People()对象能做的事，tom也能做

```java
//下面两段代码等价
new People().getclass();

People tom = new People();
tom.getclass();
```



###### 对象的销毁

每个对象都有生命周期，当对象的生命周期结束时，分配给该对象的内存地址需要被回收，在其他语言中，西药用户手动回收废弃的对象

当出现以下两种情况的时候，对象会被java虚拟机视为垃圾：

* 对象引用超出其作用范围
* 给对象赋值为null
